# Nyhetsspår：瑞典新闻学习网站设计规格

日期：2026-07-23
状态：已完成对话设计确认，等待书面规格审核

## 1. 项目目标

Nyhetsspår 是一个面向已完成 SFI D、希望通过真实瑞典新闻继续自学的个人网站。系统每天从 SVT、Aftonbladet 和 Dagens Nyheter（DN）的公开可读内容中选择 2–3 篇新闻，生成瑞典语学习材料，并在斯德哥尔摩时间每天 07:00 前后发布。

网站必须：

- 同时关注瑞典国内/本地和国际新闻；
- 在政治、经济、民生、文化、体育之间保持 7 天滚动均衡；
- 识别三家媒体对同一事件的重复报道，只保留一个主要版本；
- 不抓取或绕过付费墙、登录限制及其他访问限制；
- 不公开转载完整新闻正文；
- 为每篇生成 300–500 个瑞典语单词的学习正文；
- 不按难度筛选或简化语言，只标注文章难度；
- 用中文和英文解释生词、词组、固定搭配与语法；
- 允许用户将项目标记为“已认识”，以后不再提示其词形变化；
- 免费托管，除 OpenAI API 用量外不产生固定云服务费用。

## 2. 已确定的产品决策

| 项目 | 决策 |
| --- | --- |
| 部署 | GitHub Pages |
| 定时任务 | GitHub Actions |
| 更新时间 | Europe/Stockholm 每天 07:00，允许 GitHub 调度产生少量延迟 |
| 内容来源 | SVT、Aftonbladet、DN 的公开可读文章 |
| 每日数量 | 2–3 篇 |
| 正文长度 | 每篇 300–500 个瑞典语单词 |
| 难度 | 不设上下限；仅标注 CEFR 等级或等级区间 |
| 版权处理 | 生成原创新闻学习正文，仅引用 2–4 个短原句用于语言分析 |
| 去重 | 标题、事件实体和语义的跨来源、跨日期去重 |
| 首页布局 | “每日一期”：先显示当天 2–3 篇概览 |
| 阅读页布局 | 桌面端正文在左、语言卡片在右；移动端上下排列 |
| 用户数据 | 浏览器本地保存，不跨设备同步 |
| AI | OpenAI API，密钥保存在 GitHub Secret |
| 费用目标 | OpenAI API 每月不超过 5 美元 |

本规格中的“本地新闻”指瑞典境内任一城市或地区的区域新闻，不使用浏览器定位，也不针对用户所在城市个性化。

## 3. 明确不做的事情

首版不包含：

- 用户注册、登录或多设备同步；
- 原文全文镜像或付费内容抓取；
- 绕过 robots、登录、订阅或技术访问限制；
- 向媒体网站自动登录；
- 音频合成、口语评分或聊天导师；
- 社交分享、排行榜和多用户功能；
- 自定义域名；
- 原生 iOS 或 Android 应用。

这些功能不得为首版引入数据库、认证系统或固定云服务费用。

## 4. 系统架构

项目采用一个公开 GitHub 仓库，包含静态前端、新闻处理脚本、测试、每日课程数据和 GitHub Actions 工作流。

推荐技术栈：

- 前端：React、TypeScript、Vite；
- 样式：CSS Modules 或集中式设计 token，不依赖运行时 CSS 服务；
- 抓取与处理：Node.js、TypeScript；
- HTML 提取：Cheerio 与 JSON-LD 解析；
- AI：OpenAI Responses API 的结构化输出；
- 单元/集成测试：Vitest；
- 浏览器测试：Playwright；
- 发布：GitHub Pages。

### 4.1 主要模块

1. `source-adapters`
   - `svt`
   - `aftonbladet`
   - `dn`
   - 每个适配器只负责发现候选链接、读取公开文章元数据和提取允许访问的文本。

2. `access-filter`
   - 检查付费墙、登录跳转、`isAccessibleForFree`、页面标记、正文长度和可解析性。
   - 不可确认公开可读时一律排除。

3. `normalizer`
   - 统一来源、规范链接、标题、发布时间、栏目、作者、正文片段和结构化元数据。

4. `deduplicator`
   - 做精确、近似和事件级去重。

5. `selector`
   - 依据国内/国际范围、主题、来源、时效性和过去 7 天记录选择 2–3 篇。

6. `lesson-generator`
   - 调用 OpenAI，生成结构化学习内容。

7. `lesson-validator`
   - 检查事实来源、字数、字段、重复项目和输出格式。

8. `publisher`
   - 更新当天 JSON、历史索引和选题台账，触发静态站点构建。

9. `web-app`
   - 加载静态课程数据；
   - 在浏览器本地维护学习进度和已掌握项目。

### 4.2 首次部署条件

运行云端版本需要用户拥有：

- 一个 GitHub 账户；
- 一个公开 GitHub 仓库；
- 已启用的 GitHub Actions 与 GitHub Pages；
- 写入仓库内容和部署 Pages 所需的工作流权限；
- 一个 OpenAI API Key，保存为仓库 Secret `OPENAI_API_KEY`；
- 一个仓库变量 `OPENAI_MODEL`，未设置时使用规格规定的默认模型。

项目必须提供逐步部署说明和 `.env.example`。本地开发密钥只能放入被 Git 忽略的 `.env.local`。

## 5. 每日任务流程

### 5.1 定时策略

GitHub Actions 的 cron 使用 UTC，不能直接跟随瑞典夏令时。工作流每天在 05:00 UTC 和 06:00 UTC 各触发一次：

- 脚本用 `Europe/Stockholm` 计算当地日期和小时；
- 当当地时间早于 07:00 时退出；
- 当当天课程已经生成时退出；
- 夏令时由 05:00 UTC 的任务生成，06:00 UTC 的任务检测到已完成后退出；
- 冬令时由 05:00 UTC 的任务跳过，06:00 UTC 的任务生成；
- 支持 `workflow_dispatch` 手动补跑指定日期。

### 5.2 处理步骤

1. 从三个来源收集最近 24 小时的候选文章；
2. 标准化链接和元数据；
3. 排除付费、登录、纯视频、直播滚动、过短、无法稳定提取的文章；
4. 与当天候选及过去 7 天课程进行事件级去重；
5. 计算范围、主题、来源和信息质量分数；
6. 选择 2–3 篇；
7. 为每篇生成结构化学习材料；
8. 校验 300–500 词、双语字段、引用长度和事实来源；
9. 写入当天课程与索引；
10. 构建并发布 GitHub Pages；
11. 运行发布后冒烟检查。

## 6. 来源与访问规则

### 6.1 新闻发现

- 优先使用媒体公开提供的 RSS；
- Aftonbladet 使用其官方新闻、体育、文化等 RSS；
- 没有合适 RSS 的来源或栏目读取公开栏目页和页面公开元数据；
- 来源适配器使用固定、透明的 User-Agent；
- 每天只运行一次主要抓取，不做高频轮询；
- 每个来源设置请求间隔、超时和最大候选数。

### 6.2 公开可读判定

以下任一情况出现即排除：

- 页面要求登录或订阅；
- 页面重定向到登录/购买页面；
- 结构化数据明确表示 `isAccessibleForFree: false`；
- 来源适配器发现已知付费标记；
- 可提取正文不足以支持事实完整的 300–500 词学习文章；
- 页面主要内容是视频、音频或实时滚动，缺少稳定文字稿；
- robots 或来源访问规则不允许自动读取；
- 多次请求仍无法稳定获得内容。

系统不会尝试使用 Cookie、隐藏接口、浏览器自动登录、缓存绕过或其他方式突破限制。

## 7. 去重设计

去重目标不只是避免同一 URL，还要避免 SVT、Aftonbladet 和 DN 用不同标题报道同一事件。

### 7.1 第一阶段：确定性去重

- 规范化 canonical URL；
- 去除追踪参数；
- 规范化标题大小写、标点和常见媒体前缀；
- 比较来源文章 ID；
- 比较高度相似的标题与发布时间。

### 7.2 第二阶段：事件指纹

为每个候选生成结构化事件指纹：

- 主要人物或机构；
- 发生了什么；
- 地点；
- 时间；
- 核心结果或决定；
- 主题；
- 国内/国际范围。

如果主要实体、动作、时间和结果一致，则视为同一事件，即使标题不同。

### 7.3 第三阶段：语义复核

边界案例交由 AI 以结构化 JSON 判断：

- `same_event`；
- `confidence`；
- `reason`；
- `material_update`。

只有 `same_event: true` 且 `confidence >= 0.85` 才自动合并。低于阈值时保留为不同事件，避免误删。

### 7.4 跨日期重复

当前候选与过去 7 天的事件指纹比较：

- 没有实质更新时不重复选择；
- 出现新政策决定、新数据、新结果或明确进展时可再次选择；
- 再次选择必须标记为“后续报道”，并说明新增信息。

### 7.5 同一事件的代表文章

同一事件的多个来源按以下因素选择主要版本：

- 公开可读且提取稳定；
- 事实信息完整；
- 有足够上下文生成 300–500 词内容；
- 发布时间较新；
- 语言表达有学习价值；
- 有助于过去 7 天的来源平衡。

未选版本只保存标题、来源和原文链接，显示在“相关报道”中，不生成第二份重复课程。

## 8. 选题与 7 天均衡

每天：

- 至少 1 篇瑞典国内或本地新闻；
- 至少 1 篇国际新闻；
- 第 3 篇优先补足过去 7 天较少出现的主题。

每篇有两个独立维度：

- 范围：`local`、`sweden`、`international`；
- 主题：`politics`、`economy`、`daily-life`、`culture`、`sports`。

选择器维护过去 7 天的：

- 范围计数；
- 主题计数；
- 来源计数；
- 事件指纹；
- 后续报道记录。

主题、来源不足会提高候选分数，但不会压过公开可读、事实完整和非重复这三个硬条件。单一来源失败时允许其他来源补足，不为凑来源数量发布低质量或受限内容。

## 9. 学习内容生成

### 9.1 瑞典语学习正文

每篇生成一篇 300–500 个瑞典语单词的原创新闻学习正文：

- 只使用来源文章中可确认的事实；
- 使用自然、现代的瑞典新闻语言；
- 不按 SFI D、A2 或 B1 主动简化；
- 不人为增加高级词汇；
- 不复制来源的长段落；
- 标题、标签、双语解释和语言卡片不计入字数；
- 按空白分词并处理常见标点后计数；
- 少于 300 或多于 500 时允许一次自动修正；
- 第二次仍不合格则该篇生成失败，不发布不合格结果。

### 9.2 难度标注

难度仅作描述，不参与筛选。输出：

- CEFR 单级或区间，例如 `B1`、`B1–B2`、`C1`；
- 难度理由；
- 预计阅读时间。

难度理由可包含：

- 长复合词；
- 抽象政治或经济词汇；
- 被动态；
- 倒装；
- 长从句；
- 新闻体省略；
- 不常见固定搭配。

### 9.3 摘要与双语内容

每篇包含：

- 60 秒瑞典语摘要；
- 对应中文摘要；
- 对应英文摘要；
- 事实要点列表；
- 来源和发布日期。

默认不提供学习正文的逐句全文翻译，以保持瑞典语阅读训练。用户可通过摘要和语言卡片获得理解支持。

### 9.4 原句解析

每篇引用 2–4 个来源文章中的短句：

- 只选择有明确语言学习价值的句子；
- 单个引文最多 25 个瑞典语单词；
- 每篇全部直接引文合计最多 80 个瑞典语单词；
- 超长原句只引用相关从句并用省略号明确标识，不改写引文；
- 引用不形成原文的连续大段复制；
- 标注来源链接；
- 分析新闻体词汇、固定搭配或语法；
- 中文和英文解释必须同时存在；
- 不更改引文内容，改写内容必须与引文明确分开。

### 9.5 词汇

每个词汇项目包含：

- 原文表面形式；
- 词元；
- 词性；
- 常见变形；
- 当前语境中的中文释义；
- 当前语境中的英文释义；
- 瑞典语例句；
- 是否为复合词及其构成；
- 可选的使用提示。

### 9.6 词组与固定搭配

每个搭配包含：

- 规范形式；
- 原文形式；
- 中文释义；
- 英文释义；
- 瑞典语例句；
- 动词变化或可替换成分；
- 使用场景。

### 9.7 语法

每个语法项目包含：

- 原文片段；
- 结构名称；
- 中文说明；
- 英文说明；
- 该句中各部分的作用；
- 一个额外瑞典语例句。

语法说明重点覆盖新闻中常见的被动态、倒装、分词、从句语序、情态表达和客观报道措辞。

## 10. AI 输出与事实约束

OpenAI 调用使用结构化输出。首版默认模型为 `gpt-5.4-mini`，可通过 `OPENAI_MODEL` 仓库变量替换而不改变数据格式。模型价格和可用性在实施与部署时必须根据 OpenAI 官方模型文档再次核对。

生成提示必须：

- 附上唯一来源文章的提取文本和元数据；
- 明确禁止添加来源未支持的数字、人物、因果关系或背景事实；
- 要求不确定时省略，而不是猜测；
- 将直接引文与改写内容分开；
- 输出符合版本化 JSON Schema。

发布前执行：

- 字数校验；
- URL 与来源校验；
- 引文必须能在提取文本中逐字找到；
- 所有中文和英文字段非空；
- 词元和表面形式一致性检查；
- 当天课程内部重复检查；
- 不支持事实的启发式检查。

系统无法保证 AI 永不出错，因此页面必须显示“AI 辅助学习内容，请以原文为准”和原文链接。

- `originalSentenceNotes`

### 11.3 索引与台账

- `public/data/index.json`：历史课程日期、标题、主题和状态；
- `data/editorial-ledger.json`：过去 7 天范围、主题、来源和事件指纹；
- `data/cache/`：只保存规范链接、正文哈希、结构化元数据和已生成课程的引用；
- 缓存中不保存 API Key 或个人学习数据。

抓取到的完整新闻正文只存在于单次 GitHub Actions 运行的临时内存或临时文件中，任务结束后删除，不提交到公开仓库、不进入 Actions artifact，也不写入日志。

## 12. 浏览器本地数据

使用带版本号的 localStorage 键：

- `nyhetsspar.known.v1`
- `nyhetsspar.progress.v1`
- `nyhetsspar.settings.v1`

### 12.1 已掌握项目

每条记录包含：

- 类型：`lemma`、`phrase` 或 `grammar`；
- 规范键；
- 用户最初标记的形式；
- 标记日期；
- 可选来源文章 ID。

规则：

- 标记一个词元后，其大小写和常见词形变化不再提示；
- 固定搭配按规范形式独立记录；
- 语法点按稳定的语法标识独立记录；
- 当前文章中的标记立即淡出；
- 5 秒内提供撤销；
- “我的已掌握内容”页面可搜索并恢复；
- 恢复后未来课程重新显示；
- 支持 JSON 导出与导入；
- 导入前校验版本与字段，错误文件不覆盖现有数据。

导入默认与现有记录合并，同类型、同规范键只保留一条；导入不会删除现有记录。清空全部记录必须通过单独的确认操作完成。

### 12.2 学习进度

记录：

- 已打开文章；
- 已完成文章；
- 当日完成数；
- 最近阅读位置。

不将进度写回公共仓库，不跨浏览器或设备同步。

## 13. 页面与交互

### 13.1 今日首页

采用已确认的“每日一期”布局：

- 日期和问候；
- 今日 2–3 篇文章卡片；
- 范围、主题、来源、难度和预计时间；
- 今日完成进度；
- 生成延迟或来源故障提示；
- 历史课程入口；
- 已掌握内容入口。

### 13.2 阅读页

桌面端：

- 左侧为标题、元数据、60 秒摘要、学习正文、原句解析和原文链接；
- 右侧为当前文章的词汇、固定搭配和语法卡片；
- 点击正文标记时，右侧定位并强调对应卡片。

移动端：

- 单列布局；
- 语言卡片移动到正文之后；
- 保留下一篇、返回今日课程和完成按钮；
- 不出现横向滚动。

### 13.3 标记系统

- 词汇：黄色底线；
- 固定搭配：蓝色实线；
- 语法：紫色虚线；
- 颜色之外同时使用线型和文字标签，避免仅靠颜色表达含义；
- 所有标记可用键盘聚焦和触发；
- “我认识”按钮具有明确焦点态和撤销反馈。

### 13.4 历史课程

- 按日期倒序；
- 可按来源、范围和主题筛选；
- 不删除旧课程；
- 跨日期的后续报道显示关联标记。

### 13.5 已掌握内容

- 按词元、搭配和语法分组；
- 搜索瑞典语形式；
- 显示中文和英文释义；
- 可恢复单项；
- 提供导入、导出。

## 14. 成本控制

- 每天最多选择 3 篇；
- 抓取候选先做本地去重，再进行 AI 语义复核；
- 相同正文哈希不重复分析；
- 去重判断按批次调用，而不是每两篇单独调用；
- 网络、限流和临时服务错误最多重试 2 次，即单个操作最多尝试 3 次；
- 字数或结构错误只允许 1 次定向修正，不进入无限修正循环；
- 为请求记录输入、输出 token 和估算成本；
- 月度估算接近 5 美元时，优先减少非必要修正调用，不降低每日 2 篇的目标；
- GitHub Secret 中只保存 API Key，不写入生成数据和日志；
- 用户在 OpenAI 账户侧设置 5 美元预算或预付费限制。

## 15. 失败与恢复

### 15.1 单一来源失败

- 记录来源状态和错误分类；
- 对超时和临时服务器错误最多重试 2 次，并采用指数退避；
- 继续处理其他来源；
- 允许其他来源补足 2–3 篇；
- 下次运行重新尝试，不修改正常来源数据。

### 15.2 候选不足

- 自动重试抓取；
- 仍不足 2 篇时不使用旧新闻冒充；
- 当天 JSON 状态设为 `delayed`；
- 首页显示“今日课程生成延迟”；
- 保留历史课程正常可读；
- 允许手动补跑。

### 15.3 AI 或校验失败

- 对临时网络或速率错误指数退避重试；
- 对字数或格式错误允许一次修正；
- 单篇失败时尝试下一候选；
- 不发布缺少双语释义、来源或原文链接的文章；
- 整个任务失败时不覆盖最近一次有效数据。

### 15.4 发布失败

- 构建失败不更新 GitHub Pages；
- GitHub Actions 保留日志；
- 修复后可手动重跑同一天；
- 写入操作保持幂等，不生成同日重复课程。

## 16. 安全与隐私

- API Key 只在 GitHub Actions 服务端环境可见；
- 前端不包含调用 OpenAI 的能力；
- 日志不得输出密钥、授权头或完整环境变量；
- 外部 HTML 只作为数据解析，不直接插入页面；
- 所有展示内容经过转义；
- 外部链接使用安全的 `rel` 属性；
- localStorage 只保存学习状态，不保存敏感个人信息；
- 网站公开，因此课程 JSON、历史记录和代码均视为公开数据。

## 17. 测试策略

### 17.1 来源适配器

每个来源保存人工构造或最小化脱敏的 HTML fixture，不复制媒体完整正文。fixture 覆盖：

- 正常公开文章；
- 付费文章；
- 登录跳转；
- 纯视频或直播页面；
- 缺少正文；
- 页面结构字段缺失。

断言标题、日期、规范链接、正文、公开状态和错误类型。

### 17.2 去重与选题

覆盖：

- 同 URL；
- 不同 URL、近似标题；
- 不同标题、同一事件；
- 同人物但不同事件；
- 同一事件出现实质更新；
- 7 天内无更新的重复事件；
- 国内/国际每日最低要求；
- 五类主题 7 天滚动补足；
- 来源失败后的回退。

### 17.3 学习内容

覆盖：

- 瑞典语正文为 300–500 词；
- 难度字段存在但不作为过滤条件；
- 中文和英文摘要均存在；
- 每个语言项目同时有中英释义；
- 引文可在来源提取文本中找到；
- 不允许长段连续引文；
- JSON Schema 校验；
- 缺失字段时拒绝发布。

### 17.4 已掌握项目

覆盖：

- `regeringen` 标记后隐藏 `regering`、`regeringar` 等同词元形式；
- 不同词元不会被误隐藏；
- 固定搭配与单词记录互不覆盖；
- 语法点独立记录；
- 标记、5 秒撤销、恢复；
- 导入、导出和错误文件保护；
- localStorage 版本迁移。

### 17.5 前端与可访问性

覆盖：

- 今日首页、阅读页、历史、已掌握内容；
- 375、768、1024、1440 像素宽度；
- 键盘导航；
- 可见焦点；
- 不依赖颜色单独表达含义；
- 正文行长和字号；
- 无横向滚动；
- 数据缺失、延迟和错误状态。

### 17.6 发布冒烟测试

每次发布后检查：

- 首页返回成功；
- 当天 JSON 可读取并通过 schema；
- 当天 2–3 篇文章可打开；
- 所有原文链接格式有效；
- 静态资源无 404；
- API Key 不出现在构建产物中。

## 18. 验收标准

首版完成必须同时满足：

1. GitHub Actions 能按斯德哥尔摩时间每天约 07:00 自动运行；
2. 能从 SVT、Aftonbladet、DN 发现并过滤公开可读文章；
3. 每天正常情况下发布 2–3 篇非重复新闻课程；
4. 同一事件的跨媒体报道只生成一篇课程；
5. 选题满足国内/国际和 7 天主题均衡；
6. 每篇瑞典语学习正文为 300–500 词；
7. 难度仅标注，不限制或简化选文；
8. 生词、搭配和语法均提供中英双语解释；
9. 原句解析引用短小、可追溯且附原文链接；
10. 标记词元为“认识”后，其词形变化不再提示；
11. 已掌握内容可撤销、恢复、导出和导入；
12. 网站在手机和桌面可用；
13. API Key 不出现在前端或仓库；
14. 单一来源或单篇生成失败时系统能安全降级；
15. 月度 API 用量可观测，并以 5 美元为预算目标；
16. 自动化测试与发布后冒烟检查通过。
